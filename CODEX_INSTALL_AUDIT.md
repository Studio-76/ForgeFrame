# ForgeFrame Install Audit

Date: 2026-04-24
Host: Ubuntu/Linux environment under `/opt/ForgeFrame`

## Target-State Summary

- Normative product path is Linux host-native operation, not full ForgeFrame containerization.
- PostgreSQL is the productive primary state store; file/JSONL paths are only fallback/dev/recovery.
- Core services should be system-native, e.g. systemd-managed API, worker, retention, public HTTPS, ACME helper/renewal.
- Required runtime/API truths include `/health`, `/v1/models`, `/v1/chat/completions`, `/v1/responses`, admin/control-plane paths, queue/dispatch state, and readiness/health transparency.
- Public standard path is `https://<fqdn>/` on `0.0.0.0:443` with port `80` only as ACME/redirect helper. Missing FQDN/ACME/certificate state must be classified as a blocker or limited exception, not hidden.

## Intended Entrypoint

The intended end-to-end host operator path is `scripts/bootstrap-forgeframe.sh`.

Reasoning:
- `scripts/install-forgeframe.sh` installs artifacts and systemd units but only prints the service start commands.
- `scripts/bootstrap-forgeframe.sh` wraps the installer, validates the normative public contract, starts services, and runs `scripts/host-smoke.sh`.
- On this local system no real public FQDN or ACME email has been provided, so full public HTTPS bootstrap is expected to block unless `FORGEFRAME_BOOTSTRAP_ALLOW_LIMITED_EXCEPTION=1` is used.
